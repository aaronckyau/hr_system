# 香港中小企 HR 系統產品 Roadmap

## 目標

把目前可運行的 MVP，逐步推進為可內部試用、可驗證業務規則、可準備正式部署的香港中小企 HR / Payroll 系統。

## 現況摘要

目前已完成：

- 員工資料管理
- 請假申請與批核基礎流程
- 薪資計算、MPF、收入項目、扣款項目、離職結算第一版
- IR56B CSV / Excel 匯出 MVP
- Payslip 下載
- 員工密碼重設
- Audit Trail 第 1 版
- 主要前端文案繁體中文化

目前仍屬 MVP，主要限制在：

- 請假仍未完整支援工作天、公眾假期、半日假
- IR56B 尚未完全對齊正式申報欄位
- 權限與員工自助流程仍未完整
- SQLite 與本機啟動方式仍偏開發環境

## Phase 1：規則正確性

### 目標

先把會直接影響薪資、扣薪、報稅的核心規則做準。

### 範圍

- 請假工作天計算
- 公眾假期表
- 半日假
- 請假與扣薪整合
- MPF 特例細化
- taxable / non-taxable 對 IR56B 欄位映射
- final pay 規則補強

### 成功標準

- 請假天數不再只按曆日計算
- 未薪假與扣薪結果可追溯
- IR56B 匯出欄位與薪資資料一致

## Phase 2：正式 HR 流程

### 目標

讓系統由功能展示提升為可內部日常操作。

### 範圍

- 員工首次登入改密碼
- 忘記密碼與再次重設
- 權限模型整理：Admin / HR / Employee
- 員工主檔擴充：合約類型、在職 / 離職狀態、部門 / 職級主檔
- Payroll 鎖月與確認流程
- Audit Trail 第 2 版：篩選、查詢、匯出

### 成功標準

- 角色邊界清楚
- 已確認薪資不可任意覆蓋
- 關鍵操作皆可查核

## Phase 3：員工自助與文件

### 目標

讓系統從 HR 後台，延伸為員工也能直接使用的平台。

### 範圍

- Employee Self-Service
- 個人資料查看
- 請假申請與紀錄查詢
- 假期餘額查看
- Payslip / 報表下載
- 通知流程：請假、密碼重設、薪資單

### 成功標準

- 員工能自助完成基本操作
- HR 不需要手動分發所有文件

## Phase 4：上線準備

### 目標

收斂技術債，讓系統具備接近正式部署的條件。

### 範圍

- JWT 改為 `httpOnly cookie`
- 密碼政策與 rate limiting
- SQLite 過渡到 PostgreSQL
- migration 機制
- 測試：API / payroll / leave rule
- Docker 與 production 啟動流程
- 備份與環境設定

### 成功標準

- 可以在非本機環境穩定部署
- 核心規則有自動化測試保護

## 建議優先順序

1. 請假規則升級
2. IR56B 正式化
3. 權限與帳號流程完整化
4. Payroll 鎖月 / 確認流程
5. Employee Self-Service
6. 安全、測試、部署

## 近期里程碑

### M1：規則可信

- 請假工作天計算
- 公眾假期
- 半日假
- 請假扣薪整合
- 基本測試

### M2：流程可用

- 密碼流程完整化
- 鎖月
- Audit Trail 強化
- 角色權限整理

### M3：對外可交付

- Payslip PDF
- IR56B 正式化
- 員工自助入口
- Docker / 部署文件
