# 香港中小企 HR 系統開發 Sprint Plan

## 規劃原則

- 先處理會影響薪資正確性的底層規則
- 再補管理流程與權限
- 最後處理部署、安全與正式交付

## Sprint 1：請假規則升級

### 目標

修正目前請假只按曆日計算的限制，讓請假與薪資扣款更貼近實際。

### 工作項目

- 建立公眾假期資料模型或設定檔
- 請假天數改為工作天計算
- 加入半日假
- 請假扣薪與 payroll 重新整合
- 補對應後端測試
- 前端顯示請假明細與扣薪說明

### 完成定義

- 申請請假時可正確排除非工作天 / 公眾假期
- 半日假可申請、可批核、可影響扣薪
- 重算薪資後，扣薪金額與請假明細一致

## Sprint 2：IR56B 正式化

### 目標

把目前 MVP 匯出提升為更接近正式香港報稅欄位的版本。

### 工作項目

- 對照 `Salary.md` 與 IRD 欄位
- 補 bonus / commission / reimbursement / final pay 映射
- taxable / non-taxable 欄位整理
- 調整 CSV / Excel 欄位與標題
- 增加驗證與範例資料

### 完成定義

- 匯出欄位與系統資料模型一致
- 核心收入 / 扣款項目已能正確落入對應欄位

## Sprint 3：帳號與權限流程

### 目標

讓 Admin / HR / Employee 的帳號流程可內部試用。

### 工作項目

- 員工首次登入改密碼
- 忘記密碼 / 重新重設密碼
- 檢查角色權限邊界
- 員工僅能查看自己的請假 / 薪資 / 個人資料
- JWT 與 session 行為整理

### 完成定義

- 員工可獨立登入
- 權限不可越界讀取其他人資料

## Sprint 4：Payroll 作業流程

### 目標

把薪資結果從可重算資料，提升為受控的正式流程。

### 工作項目

- 薪資鎖月
- 已確認薪資禁止任意覆蓋
- 重算前提示受影響紀錄
- final pay 流程補強
- Audit Trail 補更多 payroll 事件

### 完成定義

- 每月 payroll 有明確狀態
- 已確認結果有保護機制

## Sprint 5：Employee Self-Service

### 目標

讓員工可直接使用系統完成基本查詢與申請。

### 工作項目

- 個人首頁
- 個人資料查看
- 請假申請與紀錄
- 假期餘額查看
- Payslip 下載
- 通知與提醒

### 完成定義

- 員工可不經 HR 完成基本操作

## Sprint 6：安全、測試、部署

### 目標

收斂技術風險，讓系統具備部署條件。

### 工作項目

- JWT 改 `httpOnly cookie`
- rate limiting
- PostgreSQL 遷移規劃
- migration 流程
- Docker 化
- 自動化測試與部署文件

### 完成定義

- 核心功能具備基本測試
- 專案可在標準環境啟動與部署

## 建議最近兩週執行內容

### Week 1

- 完成 Sprint 1 的資料模型與後端規則
- 補請假 / 扣薪測試

### Week 2

- 完成 Sprint 1 前端調整
- 開始 Sprint 2 的 IR56B 欄位對照與匯出整理
